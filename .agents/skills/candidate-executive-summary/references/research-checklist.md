# Research Checklist

Use this checklist to ensure the dossier is thorough and reproducible.

## Minimum Coverage

Aim for at least:

- 3+ independent sources for GitHub and OSS impact
- 2+ independent sources for career history
- 2+ sources for public presence (X plus one other platform if available)
- 1+ source for public speaking (or explicit "none found")
- 1+ source for notable achievements (or explicit "none found")

If this minimum cannot be met, mark confidence as `Low` and explain why.

## Source Priority

Prefer primary sources over summaries:

1. Official profiles and repositories (GitHub, personal site, company bio)
2. First-party artifacts (talk videos, slides, papers, patents)
3. Reputable publications
4. Aggregators and third-party summaries

## Disambiguation Workflow

1. Start from known handles/URLs.
2. Cross-check employer, role, city, and timeline.
3. Reject conflicting profiles unless corroborated.
4. Keep an "assumed same person because..." note for each joined identity.

## GitHub Deep-Dive

For top repositories and recent activity, capture:

- Commit volume trend (last 12-24 months)
- Maintainer behavior (reviews, issue responses, release ownership)
- Complexity indicators (architecture, systems work, reliability work)
- Evidence of collaboration quality

Avoid over-weighting raw commit counts.

## Career Timeline Build

Build a role timeline with:

- Company
- Title
- Start/end month-year
- Approximate duration
- Major outcomes
- Reasonable context notes

Short stint flag defaults:

- `Potential concern`: <18 months
- `Strong concern`: repeated <12 month stints without credible context

Context that can neutralize a short stint:

- Acquisition/integration
- Contract/consulting engagement
- Explicit scope-limited role
- Internal move tied to promotion or reorg

## Public Presence Review

For X and other public channels:

- Evaluate technical signal, clarity, and judgment
- Note signs of thoughtful disagreement and curiosity
- Separate professional substance from posting frequency

Do not include non-job-relevant personal commentary.

## Public Speaking Review

Capture:

- Topic difficulty and relevance
- Communication clarity
- Ability to explain tradeoffs and failures
- Recency and consistency of speaking

## Evidence Logging

For each key claim, keep:

- Claim
- Supporting source URL
- Confidence level
- Any alternative interpretation

This keeps final grading defensible.
