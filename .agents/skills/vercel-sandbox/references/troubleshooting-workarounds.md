# Troubleshooting and Workarounds

Use this reference when the user reports broken behavior or conflicting documentation.

| Symptom                                                                              | Likely cause                                                                                                   | What to check                                                                                                                                                               | Typical fix                                                                                                                                   |
| ------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| Sandbox dies around five minutes                                                     | The sandbox timeout was left at the default or was not extended                                                | Stable timeout docs, pricing/limits docs, installed types                                                                                                                   | Create the sandbox with a longer timeout and extend it while long-running work is active                                                      |
| `Sandbox.get({ sandboxId })` does not recover the old filesystem                     | The sandbox already stopped, or there is no valid `sandboxId`                                                  | Stable SDK docs, installed `sandbox.d.ts`, `packages/junior/src/chat/sandbox/session.ts`                                                                                    | On stable 1.x, only active sandboxes are recoverable by ID. If true durability is required, use external storage or beta persistent sandboxes |
| Docs mention `name`, persistent resume, or automatic restore, but local types do not | You are reading beta docs/changelog while the repo is still on stable 1.x                                      | `packages/junior/package.json`, installed `sandbox.d.ts`, beta changelog                                                                                                    | Align advice to the installed SDK, or upgrade intentionally before using beta-only APIs                                                       |
| `snapshot()` made the current sandbox disappear                                      | Stable snapshot creation stops the source sandbox                                                              | Installed `sandbox.d.ts`, snapshot docs, stable SDK docs                                                                                                                    | Use snapshots only for saved starting points and expect the source sandbox to stop                                                            |
| Creating from a snapshot still starts a fresh runtime session                        | That is the intended stable model: snapshots create new sandboxes from saved state                             | Snapshot docs, installed `sandbox.d.ts`                                                                                                                                     | Treat snapshots as reusable starting points, not as active-session resume                                                                     |
| Need workspaces to survive stop/timeout boundaries                                   | Stable Vercel Sandbox does not provide that durability model                                                   | Concepts docs, stable SDK docs, persistent-beta changelog                                                                                                                   | Externalize outputs, checkpoint work elsewhere, or migrate intentionally to persistent beta                                                   |
| App request timed out, but the sandbox may still be running                          | The embedding system's timeout is shorter than the sandbox timeout                                             | Consumer timeout config, sandbox timeout config, stable docs                                                                                                                | Persist sandbox identity separately so the app can reconnect to the active sandbox later                                                      |
| Turn fails even though the sandbox timeout is longer                                 | Function or Junior turn timeout fired first                                                                    | `packages/junior/src/chat/config.ts`, `packages/junior/src/chat/app/production.ts`, `packages/junior/src/chat/respond.ts`                                                   | Align function timeout, turn timeout, and sandbox timeout; persist sandbox identity before the app can be killed                              |
| Snapshot warmup exists, but repo edits still vanish between turns                    | Junior snapshots runtime dependencies, not task state                                                          | `specs/sandbox-snapshots-spec.md`, `packages/junior/src/chat/sandbox/runtime-dependency-snapshots.ts`                                                                       | Keep using snapshots for dependency warmup; preserve task state elsewhere                                                                     |
| Persistent beta docs seem to conflict with stable lifecycle docs                     | They describe different models and package generations                                                         | Installed types, stable docs, beta changelog                                                                                                                                | State explicitly which model the current environment uses before giving advice                                                                |
| Need more than the stable plan runtime ceiling                                       | Plan limit reached even with a longer timeout configured                                                       | Pricing/limits docs                                                                                                                                                         | Reduce session duration, checkpoint externally, or change plan if appropriate                                                                 |
| Next turn starts from a clean repo after a timeout in Junior                         | The consumer app died before `app_sandbox_id` was persisted, even though the sandbox may still have been alive | `packages/junior/src/chat/runtime/reply-executor.ts`, `packages/junior/src/chat/runtime/thread-state.ts`, `packages/junior/src/chat/runtime/turn-preparation.ts`            | Persist sandbox metadata earlier, before the success-only reply path                                                                          |
| A persisted sandbox ID was ignored and a fresh sandbox was created                   | Dependency profile mismatch or hinted sandbox unavailability forced recreation                                 | `packages/junior/src/chat/sandbox/session.ts`, `packages/junior/src/chat/runtime/thread-state.ts`                                                                           | Inspect dependency profile hash changes and hinted-sandbox failure path before assuming a platform bug                                        |
| Timeout checkpoints exist in specs, but the app still restarts work from scratch     | Timeout continuation is specified but not wired in the production runtime                                      | `specs/agent-session-resumability-spec.md`, `packages/junior/src/chat/respond.ts`, `packages/junior/src/chat/runtime/turn.ts`, `packages/junior/src/chat/app/production.ts` | Treat timeout continuation as unimplemented until the runtime throws/enqueues a retryable timeout path                                        |
| `VERCEL_SANDBOX_KEEPALIVE_MS` is set, but sandboxes still die                        | Keepalive only helps after a sandbox is acquired and only while it is still running                            | `packages/junior/src/chat/sandbox/session.ts`                                                                                                                               | Verify keepalive code path runs, and do not rely on keepalive to resurrect a stopped sandbox                                                  |
