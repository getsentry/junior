# Public Speaking and Technical Content Evaluation

Use this reference to evaluate publicly visible technical communication outputs.

## Scope

- Review public talks, conference/session recordings, slide decks, engineering blog posts, and technical write-ups.
- Focus on technical correctness, depth, and practical relevance.

## Minimum evidence

- Target at least 1 substantial artifact when available.
- If no public technical content exists, mark `not-assessed`.

## What to inspect

- Technical depth and correctness.
- Clarity of problem framing and tradeoffs.
- Connection to real implementation evidence.
- Consistency over time (one-off vs sustained).

## Non-evidence

- Engagement metrics alone (likes/views/follower count).
- Personal endorsements without artifact backing.

## Required output shape

- `Public speaking and technical content evaluation`: include `evidence sufficiency`, score when assessable, positives, risks, and `References`.
- `References`: include source URL + artifact pointer + why it matters.
