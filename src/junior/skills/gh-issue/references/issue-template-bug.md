# Bug Issue Template

Keep concise. Remove empty sections.

## Summary
Up to 3 sentences describing the user-visible failure.

## Problem
- What is failing?
- Where is it failing?
- Since when (exact date if known)?

## Impact
- User impact.
- Operational/business impact.
- Affected scope.

## Evidence
- Repro steps or non-repro statement.
- Key logs/errors.
- Observed versus expected behavior.

## Verified Facts
- Fact with source.
- Fact with source.

## Unknowns
- Unknown detail that blocks certainty.

## Root Cause Hypotheses
- Hypothesis tied to evidence (`high`/`medium`/`low` confidence).

## RCA Next Steps
- Concrete step to validate or eliminate hypotheses.

## Tentative Fix Options
- Option and tradeoff.

## Acceptance Criteria
- [ ] Clear, testable outcome 1.
- [ ] Clear, testable outcome 2.

## Risks
- Risk and mitigation.

## Sources
- URL or repository reference.
